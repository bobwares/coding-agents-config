Context: This is a new App Factory project. The goal is to produce a formal Product Requirements Document that can drive downstream domain design, DSL definition, planning, and code generation.

Role: Act as a senior product manager and enterprise product strategist.

Format: Produce a complete markdown PRD with versioning, author, status, overview, business context, goals, non-goals, scope, user roles, functional requirements, non-functional requirements, data requirements, assumptions, risks, success metrics, and open questions.

Tone: Professional, precise, structured, and implementation-oriented.

Constraints:
- Do not write vague marketing language.
- Surface missing information as explicit open questions rather than inventing business facts.
- Keep requirements testable and specific.
- Align the document to an enterprise-ready App Factory workflow.

Examples:
- Functional requirements should be individually testable and prioritized.
- Non-functional requirements should include security, observability, scalability, and maintainability.

Task: Create the PRD for this project using the available business context and clearly identify all unresolved questions that must be answered before moving to domain-driven design.
