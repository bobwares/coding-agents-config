# af-memory

## Purpose

Use this skill to manage the AppFactory pipeline state stored in `.appfactory/state.yaml`. This skill provides CRUD operations that enable other `af-*` skills to:

1. read current pipeline context and inputs;
2. write outputs and results;
3. track progress through the pipeline;
4. maintain state across skill invocations.

## Implementation

This skill uses `scripts/af-state.sh` at the project root which provides bash functions for state management. Source the script in any skill:

```bash
# From coding-agents-config root
source scripts/af-state.sh

# Or with AF_PROJECT_ROOT set to target project
AF_PROJECT_ROOT=/path/to/target/project source scripts/af-state.sh
```

The script requires `yq` (YAML processor) to be installed.

## State File Location

The canonical state file is:

```
./.appfactory/memory/state.yml
```

The path is resolved via `AF_PROJECT_ROOT` environment variable or defaults to current directory.

## State File Structure

The state file is generated by `af_state_init` from a project YAML. Structure:

```yaml
version: 1
kind: appfactory-project-state

project:
  id: "<project-id>"
  name: "<project-name>"
  description: "<description>"
  source_yaml: "<path-to-project-yaml>"

stack:
  profile: "<tech-stack-profile>"
  profile_path: null
  implementation_path: null

artifacts:
  prd:
    path: "ai/specs/spec-be-prd.md"
    status: null|pending|in_progress|approved
    created_at: <ISO_TIMESTAMP>
    updated_at: <ISO_TIMESTAMP>
  ddd:
    path: "ai/specs/spec-be-ddd.md"
    status: null|pending|in_progress|approved
    created_at: <ISO_TIMESTAMP>
    updated_at: <ISO_TIMESTAMP>
  dsl:
    path: "ai/specs/dsl-be-ddd.yaml"
    status: null|pending|in_progress|approved
    created_at: <ISO_TIMESTAMP>
    updated_at: <ISO_TIMESTAMP>
  plan:
    path: "ai/specs/spec-be-plan.md"
    status: null|pending|in_progress|approved
    created_at: <ISO_TIMESTAMP>
    updated_at: <ISO_TIMESTAMP>
  implementation_manifest:
    path: "ai/specs/implementation-manifest.yaml"
    status: null|pending|in_progress|approved
    created_at: <ISO_TIMESTAMP>

workflow:
  current_stage: init|prd|ddd|dsl|plan|implementation
  stages:
    init:
      status: pending|in_progress|complete|failed
      completed_at: <ISO_TIMESTAMP>
      skill_invocation: "af-project-init"
      requires: []
    prd:
      status: pending|in_progress|complete|failed
      completed_at: <ISO_TIMESTAMP>
      skill_invocation: "af-be-build-prd"
      requires: [init]
    ddd:
      status: pending|in_progress|complete|failed
      completed_at: <ISO_TIMESTAMP>
      skill_invocation: "af-be-build-ddd"
      requires: [prd]
    dsl:
      status: pending|in_progress|complete|failed
      completed_at: <ISO_TIMESTAMP>
      skill_invocation: "af-be-build-dsl"
      requires: [ddd]
    plan:
      status: pending|in_progress|complete|failed
      completed_at: <ISO_TIMESTAMP>
      skill_invocation: "af-be-build-plan"
      requires: [dsl]
    implementation:
      status: pending|in_progress|complete|failed
      completed_at: <ISO_TIMESTAMP>
      skill_invocation: "af-be-build-implementation"
      requires: [plan]

runtime:
  target_directory: "<project-root>"
  app_factory_home: "<app-factory-home>"
  bounded_contexts: []
  turn_recommendation: null

history:
  - timestamp: <ISO_TIMESTAMP>
    stage: <stage-name>
    action: started|completed|failed
    agent: "<agent-name>"
    reason: "<failure-reason>"  # only on failed
```

## Script Functions

The `scripts/af-state.sh` script provides the following functions:

### Core Functions

| Function | Description |
|----------|-------------|
| `af_state_exists` | Check if state file exists |
| `af_state_init <project_id>` | Initialize state from project YAML |
| `af_state_get <path>` | Get a value from state |
| `af_state_set <path> <value>` | Set a string value in state |
| `af_state_set_raw <path> <value>` | Set a value preserving type (null, bool, numbers, arrays) |

### Stage Management

| Function | Description |
|----------|-------------|
| `af_state_stage_complete <stage>` | Check if a stage is complete |
| `af_state_prereqs_met <stage>` | Check if all prerequisites for a stage are complete |
| `af_state_stage_start <stage>` | Mark stage as started (checks prereqs first) |
| `af_state_stage_done <stage>` | Mark stage as complete |
| `af_state_stage_fail <stage> [reason]` | Mark stage as failed |

### Artifact Management

| Function | Description |
|----------|-------------|
| `af_state_artifact_update <artifact> <status> [path]` | Update artifact status |
| `af_state_artifact_approve <artifact>` | Approve an artifact (convenience function) |

### Runtime & Navigation

| Function | Description |
|----------|-------------|
| `af_state_runtime_set <key> <value>` | Set runtime variable |
| `af_state_current_stage` | Get current workflow stage |
| `af_state_next_stage` | Get next pending stage |
| `af_state_summary` | Print state summary |

## Commands

### 1. Initialize State

Initialize state from a project YAML.

**Usage:**

```bash
source scripts/af-state.sh
af_state_init "<project-id>"
```

**Example:**

```bash
af_state_init "customer-portal"
```

This reads `$APP_FACTORY_HOME/projects/<project-id>.yaml` and creates `./.appfactory/memory/state.yml`.

### 2. Read State

Read a value at a specific path.

**Usage:**

```bash
af_state_get "<path>"
```

**Examples:**

```bash
af_state_get "project.name"             # Read project name
af_state_get "artifacts.prd.status"     # Read PRD status
af_state_get "workflow.current_stage"   # Read current stage
```

### 3. Write State

Write a value at a specific path.

**Usage:**

```bash
af_state_set "<path>" "<value>"        # String value
af_state_set_raw "<path>" "<value>"    # Preserve type
```

**Examples:**

```bash
af_state_set "project.name" "customer-app"
af_state_set_raw "runtime.bounded_contexts" '["orders", "inventory"]'
```

### 4. Update Artifact Status

Update the status of a pipeline artifact.

**Usage:**

```bash
af_state_artifact_update "<artifact>" "<status>" [path]
af_state_artifact_approve "<artifact>"
```

**Examples:**

```bash
af_state_artifact_update "prd" "approved"
af_state_artifact_approve "ddd"
```

### 5. Stage Lifecycle

Manage workflow stage transitions.

**Usage:**

```bash
af_state_stage_start "<stage>"      # Start a stage (checks prereqs)
af_state_stage_done "<stage>"       # Complete a stage
af_state_stage_fail "<stage>" "reason"  # Fail a stage
```

**Examples:**

```bash
af_state_stage_start "prd"          # Start PRD stage
af_state_stage_done "prd"           # Complete PRD stage
af_state_stage_fail "ddd" "Validation errors"
```

### 6. Check Prerequisites

Verify dependencies before starting a stage.

**Usage:**

```bash
af_state_prereqs_met "<stage>"
```

**Example:**

```bash
if af_state_prereqs_met "ddd"; then
  af_state_stage_start "ddd"
else
  echo "Prerequisites not met"
fi
```

### 7. Show Pipeline Status

Display a summary of the current pipeline state.

**Usage:**

```bash
af_state_summary
```

**Output:**

```
=== AppFactory State Summary ===
Project: customer-portal
Current Stage: ddd

Stages:
  init:           complete
  prd:            complete
  ddd:            in_progress
  dsl:            pending
  plan:           pending
  implementation: pending

Artifacts:
  prd:                      approved
  ddd:                      null
  dsl:                      null
  plan:                     null
  implementation_manifest:  null
```

## Integration with Other Skills

### Sourcing the Script

Other `af-*` skills should source the script from the coding-agents-config root:

```bash
#!/usr/bin/env bash
# Source af-state.sh from coding-agents-config/scripts/
source "$HOME/coding-agents-config/scripts/af-state.sh"

# Or if AF_SCRIPTS_DIR is set
source "${AF_SCRIPTS_DIR:-$HOME/coding-agents-config/scripts}/af-state.sh"
```

### Reading Inputs

```bash
# Read required inputs:
prd_path=$(af_state_get "artifacts.prd.path")
prd_status=$(af_state_get "artifacts.prd.status")
current_stage=$(af_state_current_stage)
```

### Writing Outputs

After generating artifacts, skills should update state:

```bash
# After generating PRD:
af_state_artifact_approve "prd"
af_state_stage_done "prd"
af_state_stage_start "ddd"
```

### Dependency Checking

Use built-in prerequisite checking:

```bash
# Before starting DDD stage:
if ! af_state_prereqs_met "ddd"; then
  echo "ERROR: Prerequisites not met for DDD stage"
  exit 1
fi
af_state_stage_start "ddd"
```

Or check individual stages:

```bash
if ! af_state_stage_complete "prd"; then
  echo "ERROR: PRD must be completed before generating DDD"
  exit 1
fi
```

## Error Handling

### State File Missing

`af_state_exists` returns false if file doesn't exist:

```bash
if ! af_state_exists; then
  echo "No state file. Run af_state_init first."
  exit 1
fi
```

### Stage Prerequisites Not Met

`af_state_stage_start` checks prerequisites automatically and returns non-zero if not met:

```bash
if ! af_state_stage_start "ddd"; then
  # Prerequisites not met - error already printed
  exit 1
fi
```

### Stage Failure

Use `af_state_stage_fail` to record failures with reason:

```bash
if ! run_ddd_generation; then
  af_state_stage_fail "ddd" "Generation failed: validation errors"
  exit 1
fi
```

## Quality Checklist

Before completing a memory operation:

1. `yq` is installed and available in PATH
2. State file exists (use `af_state_exists` to check)
3. All timestamps are ISO 8601 format (handled automatically by script)
4. Stage status values are: `pending`, `in_progress`, `complete`, `failed`
5. Artifact status values are: `null`, `pending`, `in_progress`, `approved`
6. Prerequisites are checked before starting stages
